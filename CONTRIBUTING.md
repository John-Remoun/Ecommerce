# Contributing

## Branching

- `main` — production-ready code
- `feature/*` — new features
- `fix/*` — bug fixes

## Commits

Use clear, imperative commit messages (e.g. `add cart checkout flow`).

## Pull Requests

1. Run `npm run lint`, `npm run build`, and `npm test` before opening a PR.
2. Keep PRs focused on a single concern.
3. Link related issues when applicable.
